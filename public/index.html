💡 How does a real GPT do this? (Click to expand)

A real model like GPT-2 doesn't use a simple list of words like we are. Instead, it uses a 'vocabulary' of sub-word units called tokens.

This vocabulary is built using a clever method like Byte-Pair Encoding (BPE). The process starts with a base vocabulary of single characters and then repeatedly merges the most frequently occurring pair of tokens. For example, 't' + 'h' might merge to 'th', and later 'th' + 'e' might merge to 'the'.

This is done until a desired vocabulary size is reached. For GPT-2, this resulted in a vocabulary of 50,257 tokens. These tokens include common words, sub-words (like "ing" or "trans"), and all the base characters. This allows the model to build almost any word, even ones it's never seen, and is much more efficient.

As models evolve, so do their vocabularies. The much newer GPT-4o model uses a larger and more efficient vocabulary of nearly 200,000 tokens to better understand many languages and concepts.
